Завдання 01 Мінімум.

Що може вбудований у браузер JavaScript?

  • Додавати новий HTML-код на сторінку, змінювати існуючий вміст, змінювати стилі.
  • Реагувати на дії користувача, опрацьовувати натискання миші, переміщення вказівника, натискання на клавіші клавіатури.
  • Відправляти запити через мережу до віддалених серверів, завантажувати і відвантажувати файли (так звані технології AJAX і COMET).
  • Отримувати і надсилати куки, ставити запитання відвідувачам, показувати повідомлення.
  • Запам’ятовувати дані на стороні клієнта (“local storage”), які будуть доступні в майбутніх сесіях на цьому веб-сайті.
  • Node.js підтримує функції, які дозволяють JavaScript читати/записувати довільні файли, здійснювати мережеві запити, та інше.

Завдання 02 Мінімум.

Змінні в JS

Змінна це “іменована частинка сховища”, в якій зберігаються дані. Ми використовуємо змінні, щоб зберігати дані.

Щоб створити змінну, використовуємо ключові слова const, let і var.

Іменування змінних

В JavaScript є два обмеження, які стосуються імен змінних:

  1. Ім’я має містити лише букви, цифри або символи $ і _.
  2. Перший символ не має бути числом.
  3. Є ряд імен які зарезервовні мовою і їх використання для іменування змінних не можливе.
  4. Можна використовувати будь-яку мову для іменування змінних. Такі імена дозволені, проте є міжнародна традиція використовувати англійську мову в іменах змінних

Для написання імені, яке містить декілька слів, зазвичай використовують “верблюжий регістр” (camelCase). Тобто слова йдуть одне за одним, і кожне слово пишуть із великої букви й без пробілів: myVeryLongName..

Наприклад, якщо потрібно зберігати ім’я та прізвище в змінній, ми можемо записати таким чином: 

                            
                                const userFirstName = "Ім'я користувача";
                                const userLastName = "Призвіще користувача";

                                const userFullName = userFirstName + " " + userLastName;

Але враховуючи обмеження не можна називати змінні так: 

                        
                            const користувач = "Ім'я користувача";
                            const user LastName = "Призвіще користувача";

                            const 1повне Імя Користувача = userFirstName + " " + userLastName;

Завдання 03 Мінімум.

Коментарі

В JS можливі три види коментарів:

Однорядкові коментарі починаються з подвійної косої риски //.

Багаторядкові коментарі починаються з косої риски з зірочкою /* і закінчується зірочкою з косою рискою */.

І javadoc коментар. Починається із косої риски і двох зірочок /** і закінчується зірочкою з косою рискою */. Ці коментарі дають можливість додавати в програму інформацію про неї, яка пізніше може бути використана утилітою javadoc (входить до складу Java Development Kit) для створення HTML-файлів. Коментарі документації можна використовувати при коментуванні:

Характерною рисою останього є те, що кожен новий рядок коментаря пичинається із символа *. 

                        
                            /**
                                * строка коментаря
                                * строка коментаря
                                * строка коментаря
                            */

Завдання 04 Мінімум.

Стилі написання імен змінних

Програмісти використовують численні способи, щоб забезпечити читабельність і впорядкованість коду.

Іменування змінних є важливим аспектом для того, щоб зробити код читабельним. Назви змінних слідують простій ідеї: створюйте змінні, які описують їх функції та дотримуються узгодженої теми у всьому коді. Давайте розглянемо деякі правила іменування.

  • Snakecase: Слова відокремлюються знаком підкреслення. 

                                    
                                    variable_one
                                    variable_two

  • Pascalcase:: Слова розмежовуються великими літерами. 

                                    
                                    VariableOne
                                    VariableTwo

  • Camelcase:: Слова розмежовуються великими літерами, крім початкового слова.. 

                                    
                                    variableOne
                                    variableTwo

В JS в переважній більшості випадків ми будемо використовувати camelcase нотацію для змінних
